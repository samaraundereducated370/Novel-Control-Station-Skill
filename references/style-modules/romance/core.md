# 爱情模块

爱情线不是靠“终于在一起”才成立，而是靠欲望、恐惧、自尊、照顾、误判和情感债在场景里持续摩擦。关系推进必须经由压力，而不是靠空转拖延。

## 适用时机

- 吸引和阻碍共同塑造章节
- 关系张力需要有代价地移动
- 亲密感应通过行动、克制或遗漏显形
- 本章需要改变关系温度，而不是重复同一种暧昧

## 覆盖子型

- 当代、历史、校园、职场、奇幻、科幻、悬疑混合爱情
- 慢热、敌对转爱、二次机会、禁忌关系、黑暗爱情
- 主线爱情与副线爱情

各子型都适用本模块。通用不变项是：双方必须彼此重要，障碍必须真实，关系必须移动；若爱情是主风格，还需要在 `09-style-guide.md` 中写明关系推进与情感兑现预期。

## 核心原则

- 化学反应来自具体的人物差异，不来自空泛赞美
- 最稳的爱情张力是既想靠近，又怕失去、暴露或欠债
- 多数承重关系戏最好改变一项：信任、希望、占有欲、脆弱度、边界或误会深度；较静的场景也可以通过加深既有状态来前进
- 延迟可以有，但最好带来新压力、债务、理解、确认或代价，避免只是拖
- 最能打动人的亲密往往藏在小动作、照顾、判断失手和说不出口里
- 吸引之外还要有互补、互相理解或彼此改变的理由，否则只剩表面热度
- 边界、同意和风险感要清楚，尤其在高压、禁忌或黑暗关系里

## 内化优势

- 吸收爆款作品的长处：关系回报意识强，读者能持续感到“有东西在动”
- 吸收长销类型作品的长处：人物差异稳定，关系引擎耐用，不靠同一种暧昧反复拖延
- 吸收高口碑作品的长处：情感复杂度高，亲密和伤害常常同时存在，余味不只甜也不只虐

## 风格强化提示

调用本模块时，默认把爱情写成“关系移动”，不是“标签堆积”。

- 优先让心动、克制、误判、袒护、退让或占有欲落在具体动作和话语选择里
- 优先让双方在本章都带着各自想要什么、怕什么和不愿承认什么入场
- 优先让关系推进伴随新债、新风险、新理解或新边界，而不是只升温不付代价
- 如果本章是过渡章，也可以只让关系轻微偏移，不必每章都安排高浓度名场面

## 本章优先兑现

- 关系状态出现一次可感的位移
- 一次照顾、失手、回避或袒护留下新的情感债
- 一处吸引与风险同时变得更清楚

多数章节优先兑现其中 `1-2` 项即可，不必每章都做剧烈升温或剧烈反转。

## 常见伪风格

- 把化学反应写成外貌夸奖和空泛赞美
- 一拖再拖，却没有新压力
- 误会全靠低级不沟通
- 关系戏只顾甜、虐或拉扯，不改变人物和后果

## 章节级操作

1. 明确双方此章各自想要什么、怕什么。
2. 决定本章是拉近、拉远，还是把两人拖进更危险的纠缠。
3. 让一个实际动作承担情感含义，例如帮忙、隐瞒、袒护、退让、失控。
4. 若设置误会，优先让它来自人物认知、处境和伤口，不要靠低级不沟通。
5. 结尾留下新的情感债、风险或期待，而不是原地踏步。

## 与其他风格混用

- 和 `humor` 混用时，笑点应暴露心动、防御、吃醋或亲密失衡
- 和 `suspense` 混用时，可把告白时机、关系暴露或失去对方变成风险轴
- 和 `literary` 混用时，主题和意象可以加深关系，但不能替代具体互动

## 修订检查

- 关系压力是否具体到人，而不是泛泛“很有感觉”
- 场景是否改变了关系状态
- 情绪移动是否体现在动作、遗漏或失控里
- 吸引是否始终伴随风险、代价或边界问题

## 常见失误

- 一拖再拖却没有新压力
- 把化学反应写成外貌夸奖和标签赞美
- 误会全靠低效沟通
- 关系戏只会升温，不会出现反噬和后果
