# 推理模块

推理的根不是故弄玄虚，而是证据管理。读者可以被带偏，但真相必须在回看时可追溯、可复盘、可自证。

## 适用时机

- 中心问题需要被真正调查，而不是等作者宣布答案
- 线索、可能性分支、错误解释需要同步控制
- 真相的揭晓顺序和真相本身同样重要
- 读者需要在“能猜”和“猜不透”之间维持张力

## 覆盖子型

- whodunit
- howdunit
- whydunit
- 本格、公路、程序、轻推理、黑色混合型
- 带超自然或幻想元素的调查叙事

各子型都适用本模块。通用硬约束是证据必须真实存在，结尾必须能回推，误导必须公平。

## 核心原则

- 中心问题必须具体，最好能落到谁、为什么、如何、何时中的一个或多个
- 真线索应该尽早出现，但可以被合理误读
- 红鲱鱼的作用不是作弊，而是提供一个看起来更顺手的错误解释
- 如果该子型依赖嫌疑人池或多种可能性分支，它们都要各自成立，而不是只有一个真选项像活人
- 结局必须能倒推回来，而不是靠最后一章空降信息

## 内化优势

- 吸收爆款作品的长处：中心问题够抓人，调查推进有明确回报
- 吸收长销类型作品的长处：证据公平，错误解释成立，揭晓顺序有层次
- 吸收高口碑作品的长处：真相不仅解释事件，也重写人物关系和道德评价

## 风格强化提示

调用本模块时，默认把推理写成“逐步拆旧解释”，不是“作者最后公布答案”。

- 优先让本章至少新增、重估或重排一项证据价值，不要只复述已知信息
- 优先让不同人物对同一证据产生不同解读，从而推动行动、判断和关系变化
- 优先让误导看起来是更顺手的解释，而不是明显站不住的陪跑方案
- 如果本章不适合大揭晓，也要让调查位置、嫌疑结构或真相距离发生变化

## 本章优先兑现

- 一项新证据或旧证据的新读法
- 一种合理但错误的解释被建立、加深或拆除
- 一步调查推进，或一次真相逼近带来的关系变化

多数章节优先兑现其中 `1-2` 项即可，不必每章都做重大揭晓。

## 常见伪风格

- 到最后才引入关键答案
- 线索不是被误读，而是被彻底藏掉
- 所有错误解释都明显不成立
- 一到揭晓就改成整段说明文灌输

## 章节级操作

1. 先明确本章新增或重估的证据是什么。
2. 再明确这份证据此刻最合理的误读方式是什么。
3. 让人物对证据做出不同解读，推动关系、判断和行动变化。
4. 如果本章不揭晓真相，也要推进调查位置，而不是原地兜圈。
5. 每次揭晓后都检查是否还保留足够悬而未决的调查价值。

## 与其他风格混用

- 和 `suspense` 混用时，推理控制真相路径，悬念控制逼近感和时间感
- 和 `horror` 混用时，超自然或异常现象也要遵守证据公平，不能只靠气氛顶替逻辑
- 和 `literary` 混用时，象征和主题可以加深意味，但不能模糊证据链

## 修订检查

- 真线索是否出场得足够早
- 红鲱鱼是否有其自身合理性，而不是作者硬拐
- 读者是否能在结尾回推答案
- 调查压力是否改变了人物行为和关系
- 本章是否真的让“知道更多”或“误得更深”

## 常见失误

- 到最后才引入关键答案
- 线索不是被误读，而是被彻底藏掉
- 所有主要错误解释都明显站不住，或所有嫌疑人都明显陪跑
- 一次性把全部答案用说明文倒出来
